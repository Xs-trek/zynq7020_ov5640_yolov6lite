# zynq7020_ov5640_yolov6lite

Zynq-7020 + OV5640 embedded Linux camera demo with YOLOv6Lite-S inference,
MJPEG streaming, REST/WebSocket control, V4L2-based capture/control, PS_KEY1
launcher, and A/B `image.ub` OTA support.

This repository is a public `v0.1.0` source snapshot intended for backup,
review, and reproducible rebuild on another development machine. It does not
include private board credentials, intermediate logs, Vivado project files,
PetaLinux build caches, or generated boot images.

## Baseline

- Board class: Zynq-7020, PS + PL design.
- Linux stack: PetaLinux 2022.2, Linux 5.15.36-xilinx-v2022.2.
- Camera: OV5640 8-bit DVP captured by PL and exposed to PS through V4L2.
- Application: `/usr/bin/cam_system`.
- Model: YOLOv6Lite-S NCNN assets under `assets/models/yolov6lite_s/`.
- Release tag: `v0.1.0`.

## Repository Layout

```text
src/                         userspace application source
include/cam_system/          public project headers
assets/models/               model assets packaged into rootfs
packaging/petalinux/         PetaLinux/Yocto recipe templates
scripts/build/               host-side build and validation scripts
scripts/petalinux/           recipe staging helpers

docs/                        design, package, OTA, driver and test documents
third_party/cjson/           vendored cJSON source
```

## Not Included

The following are intentionally excluded from the public repository:

- Vivado project, bitstream, HWH and XSA delivery files.
- PetaLinux `build/`, `images/`, `downloads/` and `sstate-cache` contents.
- Board-specific WiFi credentials.
- Personal SSH public keys and private keys.
- Intermediate board logs and temporary benchmark outputs.
- Generated `BOOT.BIN`, `boot.scr`, `image.ub`, `.xsa`, `.bit`, `.hwh`.

## Build Prerequisites

The VM or host should provide:

- PetaLinux 2022.2 environment.
- A configured PetaLinux project for the target XSA.
- ARM SDK environment generated by the PetaLinux project.
- Local or recipe-managed builds of NCNN, libjpeg-turbo and OpenCV as described
  in `docs/PETALINUX_PACKAGE_WORKFLOW.md` and `docs/THIRD_PARTY_COMPONENTS.md`.

For local userspace compile checks:

```sh
source ~/workdir/plnx_prj/zynq_plnx/images/linux/sdk/environment-setup-cortexa9t2hf-neon-xilinx-linux-gnueabi
./scripts/build/build_arm.sh
./scripts/build/test_ws_protocol.sh
./scripts/build/test_cmd_handler_route.sh
./scripts/build/check_project_static.sh
```

## PetaLinux Build

From the PetaLinux project:

```sh
petalinux-build
petalinux-package --boot --force \
  --fsbl images/linux/zynq_fsbl.elf \
  --fpga images/linux/system.bit \
  --u-boot
```

The regular package path installs:

```text
/usr/bin/cam_system
/usr/sbin/cam-launcher
/usr/sbin/cam-ota
/usr/share/cam-system/models/yolov6lite_s/
/usr/share/cam-system/manifest/
```

## OTA Deployment

The project OTA flow updates `image.ub` through A/B slots on the SD boot
partition. See `docs/BOOTLOADER_OTA_WORKFLOW.md`.

Typical board-side flow:

```sh
cam-ota status
cam-ota install /tmp/image.ub v0.1.0
sync
reboot
cam-ota mark-good v0.1.0
```

Only run `mark-good` after the new image boots and basic application checks pass.

## Runtime

The application listens on port `8080` after it is started by `cam-launcher` or
manually:

```sh
sudo /usr/bin/cam_system
```

Main endpoints:

```text
GET  /api/status
POST /api/camera
POST /api/yolo
POST /api/settings
POST /api/track
GET  /stream
GET  /ws
```

`POST /api/zoom` is kept as a compatibility no-op. WebSocket input is a limited
protocol subset documented in `docs/WEBSOCKET_PROTOCOL.md`.

## Security Notes

This snapshot is a lab/development baseline, not a production security design.
Do not commit WiFi credentials, private keys, device-specific SSH public keys, or
generated boot artifacts. Production use would require signed updates, stronger
device identity, access control, and a production credential process.
